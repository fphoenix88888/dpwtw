# Tocas CMS 部署與使用說明書

恭喜您！您的 **Tocas UI 全端網站管理系統** 已開發完成。這是一份完整的部署與操作指南，協助您將網站上線並開始運作。

---

## 1. 專案簡介

本專案是一個基於現代 Web 技術建構的輕量級 CMS（內容管理系統），特色如下：

*   **核心技術**：React 19 + TypeScript + Vite
*   **視覺框架**：Tocas UI 5.0 (台灣在地化 CSS 框架) + Tailwind CSS v4
*   **資料儲存**：瀏覽器 LocalStorage (純前端模擬資料庫，易於部署至靜態空間)
*   **主要功能**：
    *   **RWD 響應式設計**：完美支援手機、平板與桌面。
    *   **完整後台管理**：文章、頁面、分類、媒體庫、使用者與權限管理。
    *   **站長日誌**：支援置頂排序、標籤雲與全文搜尋。
    *   **活動日曆**：直覺的月曆視圖與活動管理。
    *   **系統備份還原**：一鍵匯出/匯入 JSON 資料，輕鬆轉移。
    *   **安裝導引 (Setup Wizard)**：初次使用自動引導設定。

---

## 2. 如何部署 (Deployment)

由於本專案是純靜態網站 (Static Web App)，您可以將其部署到任何支援靜態網頁的伺服器或託管服務，無需設定後端資料庫。

### 方法 A：使用 GitHub Pages / Vercel / Netlify (推薦)

1.  將專案原始碼上傳至您的 GitHub Repository。
2.  連結 Vercel 或 Netlify，並設定：
    *   **Framework Preset**: Vite
    *   **Build Command**: `pnpm build` (或 `npm run build`)
    *   **Output Directory**: `dist`
3.  部署完成後，您將獲得一個專屬網址。

### 方法 B：傳統網頁伺服器 (Apache/Nginx)

1.  在本地端執行建置指令：
    ```bash
    npm install  # 安裝依賴
    npm run build # 打包專案
    ```
2.  將生成的 `dist/` 資料夾內的 **所有檔案** 上傳至伺服器的公開目錄 (public_html)。
3.  **注意**：請確保您的伺服器設定了 URL Rewrite 規則，將所有請求導向 `index.html`，以支援 SPA 路由 (本專案預設使用 Hash Router `/#/`，若伺服器未設定 Rewrite 也能直接運作，相容性最高)。

## 3. 初次使用設定 (Setup)

當您將網站部署完成並首次開啟首頁時，系統會自動偵測並進入 **「安裝導引模式」**：

1.  **進入安裝頁面**：瀏覽器將自動跳轉至 `/setup`。
2.  **設定網站資訊**：輸入您想要的「網站名稱」。
3.  **建立管理員**：設定第一位超級管理員 (Super Admin) 的名稱、Email 與密碼。
4.  **完成安裝**：點擊「完成安裝」後，系統會自動登入並進入後台儀表板。
    *   *注意：安裝完成後，網站預設處於「維護模式」，前台訪客暫時無法瀏覽，直到您手動開啟。*

## 4. 後台管理指南

後台入口通常為 `您的網址/#/admin/login` (若已登入可直接點擊導覽列進入)。

### 基礎操作
*   **網站設定**：前往「網站設定」頁面，您可以：
    *   上傳 **Logo** 與 **Favicon**。
    *   修改首頁 **Hero 區塊** 的標題與描述。
    *   自訂 **頁尾 (Footer)** 的內容、連結與配色。
    *   **關閉維護模式**：當內容準備好後，請在此將「維護模式」關閉，網站即正式對外開放。

### 內容管理
*   **站長日誌**：撰寫技術文章或隨筆，支援 Markdown 語法與圖片插入。使用「排序置頂」功能可調整精選文章順序。
*   **頁面管理**：建立「關於我們」、「服務項目」等單頁。支援階層式目錄（設定母頁面），前台會自動生成下拉選單。
*   **媒體庫**：批次上傳圖片，並在編輯文章時直接插入。

### 系統維護
*   **備份資料**：定期前往「系統資訊」頁面，點擊 **「備份資料」** 下載 JSON 檔。
*   **還原資料**：若更換電腦或瀏覽器，可使用「還原資料」功能上傳 JSON 檔，即可恢復所有網站內容。

## 5. 開發者指南 (Developer Notes)

若您希望自行修改程式碼：

*   **環境需求**：Node.js 18+
*   **安裝依賴**：`npm install` 或 `pnpm install`
*   **啟動開發伺服器**：`npm run dev`
*   **專案結構**：
    *   `src/pages`：所有頁面路由元件。
    *   `src/components`：共用 UI 元件。
    *   `src/services/db.ts`：模擬資料庫邏輯與種子資料 (Seed Data)。
    *   `src/index.css`：全站樣式與 Tocas UI / Tailwind 設定。

祝您使用愉快！如有任何問題，歡迎參考原始碼中的註解。
